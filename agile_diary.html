Project Description:

What is RSSOwl?

“Is an a pplications that collect data from RSS-compliant sites are called RSS readers or "aggregators." RSSOwl is such an application. RSSOwl lets you gather, organize, update, and store information from any compliant source in a convenient, easy to use interface, save selected information in various formats for offline viewing and sharing, and much more. It's easy to configure and the best of all: It's platform-independent.” .

It is written in Java , and released under the Eclipse Public License , RSSOwl began as small project on Sourceforge at the end of July 2003. RSSOwl 1.0 was released on December 19, 2004. RSSOwl was the Sourceforge Project of the Month for January 2005 and by the end of 2009 the major release version 2.0 was announced and is now available to all users.

Our Contribution

In RSSOwl we are going to edit few features through the Addition, Removal and Modification which enhances the quality of Software, providing the user a more powerful and useful tool.

  • Feature Addition
    • Connect with the facebook account of the user and subscribe to facebook's friends news feed (Notification, Friends Status Updates, Links) . A1
    • Read the emails from a Gmail account . A2
  • Feature Modification
    • Add the Facebook “like” button to each item of a channel or to a channel itself. M1
    • Give the user the possibility to directly listen to music into an rss item (f.i. podcast subscriptions with enclosures ) using a flash-based open source player and adding it at the end of the item description. M2
    • Modify the preference panel of RSSOwl putting into it some options related to our features. M3
  • Feature Deletion
    • Delete the “Mark as read” ,” Mark as Undread” functions and menu entries. D1

Description of the acceptance Tests :

  • The user can browse the facebook page associated with our application after we provided the link. (U.S. ID : 1)

  • Clicking the Facebook button on the RSSOwl's toolbar a Dialog will show up, and the user can start the authorization procedure clicking the "OK" button. (U.S. ID : 2)

  • After clicking the "OK" button in the Facebook Dialog, the user will be asked to insert fb username and password and allow the RSSOwl fb application to access his information. (U.S. ID : 2)
  • If the user gives to the facebook application permissions, clicking the "Allow" button, the procedure to get the users feed will automatically start and after a few seconds he can see on the side bar a new folder called as Facebook feeds with feeds connected to his account. (U.S. ID : 3)

  • If the user has already gave fb permissione and got the feeds ,when he click again the Facebook button, nothing will happens.(U.S. ID : 2)

  • If the user has given proper facebook authentication after clicking the save button he can see on the side bar a new folder called as Facebook feeds with feeds connected to his account. (U.S. ID : 3)

  • Clicking the Gmail button on the RSSOwl's toolbar the user will be asked to insert username and password for their gmail credential in the forms and save it using the save button . ( U.S. ID : 4 )

  • If the user has already saved the data ,when he click again the Gmail button he will see the username and the masked password that he entered before. ( U.S. ID : 4 )

  • If the user has given proper gmail authentication after clicking the save button he can see on the side bar a new folder called as Gmail with feed connected to his gmail account. ( U.S. ID : 5 )

  • Providing a valid URL that points out to an mp3 file, the user will be able to play that file within an HTML page ( given by us ) with the open source player .( U.S. ID : 6 )

  • When a rss item has a rss enclosure with the mp3 extention, if the client is connected to the Internet,the user can play the song within RSSOwl ( U.S. ID : 7 )

  • When the user has already entered his valid facebook credential,after clicking the facebook like button integrated in the item, he will see the activity related to this action on his facebook wall. ( U.S. ID : 8 )

  • The user will not be able to mark a message as read or unread. ( U.D. ID : 9 )

  • The user can edit some preferences about new implemented features and see the effects of changement after restarting the program. ( U.D. ID : 10 )

Gap Analysis:

For the first sprint we had to deal with a lot of things that were new to both of us: create a facebook application, work with Eclipse RCP application, studing coverage and unit test.This is the reason why we stimated a big amount of hours and in the end it turned out that we were right, as we ecountered several problems.
First of all the rssowl web site for developers is not up-to-date, bugzilla is not working, no one is using the forum anymore and the documentation is not exhaustive. That's why we lost some time in finding how to import and run the project properly into eclipse environment ( at the beginning we were triyng with the wrong version of eclispse ).
Then we've found out that an official library for interface java with facebook does not exists so we had to search for a valid and updated open source library.
We overstimated the time we needed to get a developer account and build a facebook application as none of us had never made it before in the past, but then it turned out to be a very trivial task.

Gap Analysis:

As soon as we started working on this User Story, we'found out that in the meantime facebook had changed their policies to access RSS feeds url. As facebook is interested to let the user visit their pages for commercial reasons and for this purpose is trying hard to keep the source of the RSS feed hidden.
So the plan we had in the beginning to retreive the URL connected to user feed got changed completely:we are not using the login dialog anymore but instead the official JS page loaded into internal browser.
Then we created a small 'walkaround' to generate the feed's URL, obtaining the keys out of the HTML source of two pages ( notifications.php and notes.php) where they are stored. This took us some extra time to achieve the result.
As we waste a lot of time literally fighting for getting the feed's URL, we splitted the task 3.4 : "Adding the rss channels" in two parts: in the first part we generated the URL, and in the second (3.5) part we will (Sprint 4) create the folder and the channels in the UI.

Gap Analysis:

RSSOwl supports both Atom and RSS (1 and 2 version) standards which can support enclosures with different markups: for this reason we spent more time than expected how to implement a unique solution. The process of identification of the mime type of mp3 in both rss and atom parsers was easier than expected as both parsers extends a basic parser class.
For creating the HTML page we took some more time as we wanted to do it using javascript that does not provide a standard way for reading http GET parameters. We had to create a external js file for that purpose.
